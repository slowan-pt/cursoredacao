## Arquivos Preparados

- `src/storage.ts`: interface de storage privado, provider local de desenvolvimento e implementação R2.
- `src/uploads.ts`: valida data URL por MIME, magic bytes e tamanho; devolve bytes validados para storage privado.
- `src/routes/aluno.ts`: grava novas redações/edições em storage privado quando a flag R2 estiver ativa.
- `src/routes/admin.ts`: hidrata arquivo privado para data URL ao abrir a correção no painel do professor.
- `src/types.ts`: binding opcional `R2_UPLOADS`.
- `wrangler.jsonc`: binding preparado para `R2_UPLOADS`.
- `migrations/004_storage_files.sql`: tabela de metadados para arquivos privados.
- `.env.example`: variável informativa `R2_UPLOADS_BUCKET=redacao-uploads`.

## Compatibilidade com o Editor Atual

- O banco passa a guardar `arquivo_url` como referência interna `r2:<object_key>` quando R2 está ativo.
- As rotas de detalhe da redação convertem a referência para data URL em runtime para preservar o editor atual do professor e a visualização do aluno.
- Essa conversão é temporária e limitada pelo `MAX_UPLOAD_BYTES`; a evolução recomendada é servir os arquivos por endpoint autenticado/streaming.
- Em desenvolvimento/homologação com `ENABLE_R2_UPLOADS=false`, o fluxo antigo base64 pode continuar sendo usado para facilitar testes locais.
- Em produção com `ENABLE_R2_UPLOADS=false`, novos uploads continuam bloqueados.

## Modelo de Chave

```text
tenants/{siteId}/turmas/{turmaId}/alunos/{alunoId}/redacoes/{correcaoId}/{fileId}.{ext}
```

A chave ajuda na organização, mas não deve ser usada como única proteção.
Toda leitura, download ou exclusão precisa validar autorização no banco.

## Tipos Permitidos

- PDF: `application/pdf`
- JPEG: `image/jpeg`
- PNG: `image/png`

O tamanho máximo continua controlado por `MAX_UPLOAD_BYTES`.

## Ativação Manual Futura

1. Bucket privado no Cloudflare R2 criado:

```bash
npx wrangler r2 bucket create redacao-uploads
```

2. Confirmar binding no `wrangler.jsonc`:

```jsonc
"r2_buckets": [
  {
    "binding": "R2_UPLOADS",
    "bucket_name": "redacao-uploads"
  }
]
```

3. Migration `004_storage_files.sql` aplicada no Supabase após backup de metadados do schema.
4. `ENABLE_R2_UPLOADS=true` configurado em `wrangler.jsonc` para deploy do Worker.
5. Testar upload e download autenticados.
6. Manter bucket sem acesso público.

## Modo Local Temporário

- Quando `APP_ENV` não é `production` e `ENABLE_R2_UPLOADS=false`, `getPrivateStorage` usa um provider local em memória.
- Esse provider serve apenas para testes de integração durante desenvolvimento.
- Os arquivos somem quando o Worker/processo local reinicia.
- Esse modo não substitui R2 e não deve ser usado para dados permanentes.

## Pendências de Integração

- Criar endpoint autenticado para download/stream e reduzir a hidratação por data URL em runtime.
- Validar:
  - aluno só acessa seus próprios arquivos;
  - professor só acessa arquivos do seu site/turmas;
  - corretor só acessa arquivos direcionados;
  - superadmin segue regra explícita.
- Criar script de migração dry-run para arquivos antigos em base64.

## Regras de Segurança

- Não confiar no nome original do arquivo.
- Não criar URL pública permanente.
- Não expor `object_key` como autorização.
- Não registrar conteúdo de arquivos em logs.
- Não tratar o provider local em memória como armazenamento persistente.
- Não migrar base64 automaticamente.
- Não ativar R2 em produção sem bucket, migration e testes.
